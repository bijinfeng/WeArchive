
# PRD：桌面端微信聊天记录备份工具

版本：v0.2
目标平台：macOS / Windows 桌面端
当前阶段：高保真原型与交互定义
产品目标：让完全不懂备份、数据库、迁移包的普通用户，也能按页面提示完成微信聊天记录的备份、查看、导入、导出、迁移和恢复。

---

## 1. 产品一句话说明

这是一个桌面端微信聊天记录备份工具。

用户打开后，不需要理解“库、索引、入库、映射、校验”等专业概念，也应该能清楚知道：

- 我现在有没有备份成功。
- 我要备份应该点哪里。
- 我要找聊天记录应该怎么搜。
- 我要导出聊天记录应该先选什么。
- 我要从旧电脑导入记录应该怎么做。
- 如果失败了，是哪里失败、影响什么、下一步怎么处理。

产品必须把复杂的数据处理流程包装成清晰、可操作、可救援的任务流。

---

## 2. 目标用户

### 2.1 普通个人用户

特点：

- 只知道“我要保存微信聊天记录”。
- 不理解数据库、校验、索引、迁移包。
- 害怕误删、覆盖、丢失数据。
- 希望一步一步跟着界面走。

核心诉求：

- 能备份。
- 能找回。
- 能导出。
- 出错时知道怎么办。

### 2.2 重度个人用户

特点：

- 微信记录很多。
- 有多台电脑或多个微信账号。
- 需要长期归档、迁移、整理。

核心诉求：

- 备份状态清楚。
- 搜索快。
- 导出可控。
- 多任务可暂停、继续、重试。

### 2.3 客服、运营、法务归档用户

特点：

- 聊天记录可能用于归档、审计、证据整理。
- 更关心完整性、时间范围、附件、报告、脱敏和加密。

核心诉求：

- 可选择导出范围。
- 可生成报告。
- 可校验完整性。
- 可控制敏感信息。

## 3. 产品设计总原则

### 3.1 任何页面都要回答三个问题

用户进入任意页面，第一眼必须知道：

1. 我现在在哪。
2. 我现在能做什么。
3. 下一步应该点哪里。

### 3.2 所有主按钮都必须有明确条件

不允许出现用户不知道为什么不能点的按钮。

按钮禁用时必须给出原因，例如：

- 请选择要导出的聊天记录。
- 请先选择保存位置。
- 密码不一致，请重新输入。
- 当前文件无法读取，请重新选择。

### 3.3 长流程不能只靠 Toast

备份、导入、导出、恢复都是长流程。

这些流程必须在页面内展示：

- 当前步骤。
- 总进度。
- 正在处理什么。
- 能否暂停。
- 能否取消。
- 失败后怎么处理。
- 已完成部分是否保留。

Toast 只能做轻量反馈，不能替代流程状态。

### 3.4 专业词必须翻译成用户能懂的话

| 内部概念 | 页面推荐文案 |
| --- | --- |
| 当前库 | 当前备份位置 |
| 入库 | 保存到当前备份位置 |
| 解析 | 读取备份内容 |
| 校验 | 检查文件是否完整 |
| 映射 | 确认保存到哪个账号 |
| 迁移包 | 可用于换电脑恢复的备份包 |
| 索引 | 用于快速搜索的记录目录 |
| 冲突 | 已有相同聊天记录 |
| 目标路径 | 保存位置 |
| 源路径 | 文件来源 |

页面可以在详情里保留专业词，但主流程文案必须使用小白可理解的说法。

## 5. 全局外壳

### 5.1 标题栏

标题栏包含：

- macOS / Windows 窗口控制区。
- 侧栏折叠按钮。
- 全局搜索框。
- 当前连接状态。
- “立即备份”主按钮。
- macOS / Windows 预览切换。

交互要求：

- 侧栏折叠按钮必须真实改变左侧宽度。
- 路由切换后侧栏折叠状态保持。
- Windows 模式不显示重复应用图标。
- 标题栏不放面包屑，避免挤占空间。
- “立即备份”点击后必须创建备份任务，不允许只弹提示。

### 5.2 左侧导航

每个导航项包含：

- 图标。
- 页面名称。
- 可选数量角标。
- 当前选中状态。

交互要求：

- 点击导航项立即切换内容区。
- 当前页面高亮。
- 切换页面时不重置侧栏折叠状态。
- 切换页面时不丢失正在执行的任务。
- 如果当前页面有未保存配置，切换前需要轻提示或保留草稿。

### 5.3 底部当前备份位置

底部“当前库”不能做成大卡片，应压缩成状态条。

默认展开态高度不超过 56px。

展示内容：

- 图标。
- 备份位置名称，例如：主力账号备份。
- 空间占用，例如：128.6GB。
- 健康状态点：正常、需检查、有错误。

悬停展示：

- 完整路径。
- 最近检查时间。
- 可用空间。
- 数据完整度。

点击后打开“备份位置详情抽屉”。

折叠侧栏时：

- 只显示数据库图标。
- 保留健康状态点。
- 悬停显示 tooltip。

顶部搜索框始终可见。

| --- | --- |

## 7. 首页

### 7.1 页面目标

让用户快速判断：

- 当前是否已经备份。
- 最近有没有失败任务。
- 数据是否完整。
- 下一步最应该做什么。

### 7.2 首屏结构

首页首屏包含：

1. 顶部状态摘要。
2. 当前微信账号卡。
3. 数据概览。
4. 最近任务。
5. 异常提醒。
6. 快捷操作。

### 7.3 顶部状态摘要

状态分为三种。

正常：

- 标题：备份状态正常
- 说明：最近一次备份已完成，当前没有需要处理的问题。
- 主按钮：立即备份
- 次按钮：查看备份记录

需要处理：

- 标题：有 2 个问题需要处理
- 说明：部分附件没有备份成功，建议重新检查。
- 主按钮：查看问题
- 次按钮：稍后处理

首次使用：

- 标题：还没有备份微信聊天记录
- 说明：请先选择微信账号并开始第一次备份。
- 主按钮：开始第一次备份
- 次按钮：了解会备份哪些内容

### 7.4 当前账号卡

展示：

- 头像。
- 昵称。
- 微信号。
- 连接状态。
- 最近备份时间。
- 自动备份开关。
- 数据完整度。

按钮：

- 重新扫描。
- 查看详情。
- 立即备份。

点击“重新扫描”后：

- 按钮变成“正在扫描”。
- 显示扫描进度。
- 显示当前扫描位置。
- 扫描完成后刷新数据。
- 扫描失败时显示失败原因和“查看日志”。

### 7.5 数据概览

KPI 包含：

- 已备份会话。
- 消息数量。
- 附件数量。
- 当前占用空间。
- 今日新增。
- 待处理问题。

交互：

- 异常数字用状态色，但不使用夸张警告视觉。

## 8. 聊天记录

### 8.1 页面目标

用户可以完成：

- 找到某个联系人或群。
- 搜索某段聊天内容。
- 查看附件。
- 导出指定聊天。
- 重新备份某个会话。

### 8.2 页面结构

使用三栏结构：

1. 左栏：会话列表。
2. 中栏：消息预览。
3. 右栏：会话详情。

窗口宽度不足时：

- 三栏改为列表 + 详情切换。
- 右侧详情可变为抽屉。
- 不允许内容互相挤压到不可读。

### 8.3 会话列表

每条会话展示：

- 头像。
- 名称。
- 类型：个人、群聊、公众号、企业微信。
- 最后一条消息摘要。
- 更新时间。
- 消息数量。
- 附件标识。
- 备份状态。
- 收藏状态。

交互：

- 单击会话：切换中间消息预览。
- 双击会话：打开会话详情视图。
- 悬停会话：显示收藏、导出、更多按钮。
- 勾选会话：进入批量选择状态。
- 多选后顶部显示批量操作条。
- 搜索命中联系人或摘要时高亮。

批量操作条包含：

- 已选择数量。
- 导出。
- 重新备份。
- 添加标签。
- 取消选择。

### 8.4 消息预览

支持消息类型：

- 文本。
- 图片。
- 视频。
- 语音。
- 文件。
- 链接。
- 小程序。
- 转账记录。
- 系统消息。

交互：

- 当前状态。
- 进度。
- 当前步骤。
- 开始时间。

### 9.3 任务状态

| 状态 | 用户看到的文案 | 可用操作 |
| --- | --- | --- |
| 等待中 | 等待开始 | 取消、提前开始 |
| 扫描中 | 正在查找聊天记录 | 暂停、查看详情 |
| 备份中 | 正在备份聊天记录 | 暂停、查看详情 |

| 已取消 | 已取消 | 删除记录 |

### 9.4 任务按钮

暂停：

- 点击后按钮立即变为“继续”。
- 进度条停止动画。
- 状态变为已暂停。
- 页面提示：已完成部分会保留。

继续：

- 点击后按钮变为“暂停”。
- 状态恢复为处理中。
- 继续从上次位置处理。

取消：

- 必须二次确认。
- 确认文案：确定取消这个任务吗？已完成的备份会保留，未完成部分不会继续。
- 主按钮：取消任务。
- 次按钮：继续等待。

重试：

- 保留原任务范围和保存位置。
- 如果失败原因是权限或磁盘空间，需要先引导用户修复。

### 9.5 任务详情抽屉

内容：

- 当前步骤。
- 总进度。
- 已处理消息数。
- 已处理附件数。
- 当前处理路径。
- 速度。
- 预计剩余时间。
- 警告数量。
- 错误数量。
- 日志列表。

日志支持：

- 全部。
- 信息。
- 警告。
- 错误。
- 复制错误信息。

## 10. 导入导出总设计

### 10.1 页面目标

导入导出页面要解决两个问题：

- 导入：把外部聊天记录保存到当前备份位置。
- 导出：把当前聊天记录保存成用户可查看、可发送或可迁移的文件。

页面不能设计成两个简单按钮，必须设计成“流程工作台”。

### 10.2 页面结构

导入导出页面由五块组成：

1. 顶部模式切换：导入 / 导出。
2. 左侧步骤条：告诉用户当前走到哪一步。
3. 中间配置区：当前步骤要用户操作的内容。
4. 右侧预览与风险区：告诉用户选择会带来什么影响。
5. 底部任务队列：展示正在执行和等待的任务。

### 10.3 模式切换

默认进入“导入”模式。

用户点击“导出”：

- 页面切换导出步骤。
- 保留导入草稿。
- 不清空正在执行的导入任务。
- 右侧预览更新为导出说明。
- 主按钮根据导出完成度变化。

用户再切回“导入”：

- 回到上次导入步骤。
- 已选文件、策略和校验结果保留。

### 10.4 底部任务队列

任务队列始终显示，除非完全没有任务。

队列项展示：

- 类型：导入或导出。
- 名称。
- 当前阶段。
- 进度。
- 状态。
- 剩余时间。
- 操作按钮。

可用操作：

- 暂停。
- 继续。
- 取消。
- 重试。
- 查看详情。
- 打开结果。
- 查看报告。

## 11. 导入流程

### 11.1 导入用户故事

用户可能想做这些事：

- 我换电脑了，想把旧电脑的聊天记录导进来。
- 我有一个以前导出的备份文件，想恢复查看。
- 我有一个数据库或目录，不知道能不能导入。
- 我只想先检查文件，不想马上写入。

导入流程必须让用户一步一步完成，不要求用户理解文件结构。

### 11.2 导入步骤

导入分为 5 步：

1. 选择来源。
2. 选择文件或目录。
3. 检查文件。
4. 确认保存方式。
5. 开始导入。

左侧步骤条显示：

- 当前步骤。
- 已完成步骤。
- 有警告步骤。
- 有错误步骤。
- 未开始步骤。

### 11.3 第 1 步：选择来源

页面标题：你要从哪里导入聊天记录？

来源卡片：

| 来源 | 卡片说明 | 主按钮 |
| --- | --- | --- |
| 本地备份文件 | 适合导入之前导出的备份文件 | 选择备份文件 |
| 旧电脑迁移包 | 适合从另一台电脑搬过来 | 选择迁移包 |
| 历史数据库 | 适合导入旧版本备份数据 | 选择数据库 |
| iPhone 备份 | 适合从 Finder 或 iTunes 备份中读取 | 选择备份目录 |
| Android 本地目录 | 适合从安卓手机导出的目录读取 | 选择目录 |

选中卡片后：

- 卡片高亮。
- 右侧说明更新。
- 主按钮从“请选择来源”变为对应动作。
- 下一步提示出现：选择后我们会先检查文件，不会立刻修改当前数据。

禁用状态：

- 没有选择来源时，主按钮禁用。
- 禁用提示：请先选择导入来源。

### 11.4 第 2 步：选择文件或目录

用户点击主按钮后，模拟打开文件选择器。

选择成功后页面展示：

- 文件名或目录名。
- 文件大小。
- 修改时间。
- 来源类型。
- 是否可读取。
- 预计检查时间。

如果选择多个文件：

- 显示文件清单。
- 显示总大小。
- 显示不支持文件数量。
- 支持移除单个文件。

按钮状态：

- 未选择文件：请选择文件。
- 已选择文件：开始检查文件。
- 文件不可读：重新选择文件。
- 正在读取：正在读取文件。

拖拽交互：

- 拖入页面时显示高亮边框。
- 拖入支持文件时显示：松开后添加文件。
- 拖入不支持文件时显示：这个文件格式暂不支持。
- 松开后文件进入清单。

错误示例：

标题：无法读取这个文件
说明：文件可能被移动、损坏，或当前账号没有读取权限。
主操作：重新选择文件
次操作：查看支持的文件格式

### 11.5 第 3 步：检查文件

检查文件就是原来的“解析 + 校验”，但页面主文案使用“检查文件”。

检查阶段展示：

- 总进度。
- 当前正在检查的文件。
- 已识别账号数。
- 已识别会话数。
- 已识别消息数。
- 已识别附件数。
- 警告数。
- 错误数。

检查子阶段：

1. 正在读取文件。
2. 正在识别账号。
3. 正在读取聊天记录。
4. 正在读取图片、语音和文件。
5. 正在检查是否完整。
6. 检查完成。

检查中按钮：

- 暂停检查。
- 查看详情。

点击暂停：

- 状态变为已暂停。
- 主按钮变为继续检查。
- 说明：已读取的内容会保留。

点击继续：

- 从暂停位置继续。
- 状态恢复为检查中。

检查完成后：

无问题：

- 标题：文件检查通过。
- 说明：可以继续导入。
- 主按钮：下一步，确认保存方式。

有警告：

- 标题：发现 6 个需要注意的问题。
- 说明：这些问题不会阻止导入，但可能影响部分附件或消息。
- 主按钮：继续导入。
- 次按钮：查看警告。

有错误：

- 标题：发现 2 个必须处理的问题。
- 说明：当前文件无法继续导入，请先处理错误。
- 主按钮：重新选择文件。
- 次按钮：导出检查报告。

### 11.6 第 4 步：确认保存方式

页面标题：这些聊天记录要保存到哪里？

用户需要确认：

- 保存到哪个备份位置。
- 保存到哪个微信账号。
- 遇到已有聊天记录怎么办。
- 附件怎么处理。

保存位置：

- 默认选中当前备份位置。
- 可点击“更换位置”。
- 更换前检查权限和空间。

账号匹配：

- 如果文件中的微信号和当前账号一致，显示绿色通过。
- 如果不一致，显示黄色提醒：这份记录看起来属于另一个微信账号。
- 用户可选择：保存到原账号、保存到当前账号、创建新账号记录。

重复记录处理：

| 策略 | 给用户的说明 |
| --- | --- |
| 跳过已存在的记录 | 最安全，已有内容不会被改动 |
| 只补充缺失内容 | 推荐，可补全缺少的附件或消息 |
| 保留两份 | 适合不确定是否重复时使用 |
| 覆盖已有记录 | 风险较高，会替换当前已有内容 |

默认策略：只补充缺失内容。

选择“覆盖已有记录”时：

- 必须二次确认。
- 用户需勾选：我知道这可能会替换已有聊天记录。
- 主按钮才可点击。

附件处理：

| 策略 | 说明 |
| --- | --- |
| 全部导入 | 会占用更多空间，但记录最完整 |
| 只导入图片和文件 | 跳过视频和语音，节省空间 |
| 跳过大文件 | 可设置大小上限 |
| 暂不导入附件 | 只保存文字消息 |

右侧预览必须实时更新：

- 将导入多少个会话。
- 将导入多少条消息。
- 将导入多少个附件。
- 预计占用空间。
- 会跳过多少重复内容。
- 有哪些风险。

### 11.7 第 5 步：开始导入

主按钮：开始导入到当前备份位置。

- 失败数。
- 剩余时间。

暂停导入：

- 主按钮变为继续导入。
- 文案：已完成部分会保留。

取消导入：

- 必须二次确认。
- 让用户选择是否保留已导入内容。
- 选项：保留已导入内容、删除本次已导入内容、暂不取消。

导入完成：

标题：导入完成
说明：已导入 18 个会话、42,816 条消息、3,240 个附件。
主操作：查看导入的聊天记录

辅助操作：继续导入其他文件

次操作：重试失败项

失败：

## 12. 导出流程

### 12.1 导出用户故事

用户可能想做这些事：

- 把某个联系人聊天记录导出来。
- 把一段时间的聊天记录保存成 PDF。
- 把所有聊天记录归档到硬盘。
- 生成可换电脑恢复的加密迁移包。
- 导出前隐藏手机号、微信号等敏感信息。

导出流程必须让用户先选范围，再选格式，再选内容，最后选择保存位置。

### 12.2 导出步骤

导出分为 5 步：

1. 选择导出范围。
2. 选择导出格式。
3. 设置导出内容。
4. 选择保存位置。
5. 开始导出。

### 12.3 第 1 步：选择导出范围

页面标题：你要导出哪些聊天记录？

范围选项：

| 范围 | 说明 |
| --- | --- |
| 全部聊天记录 | 导出当前备份位置里的全部内容 |
| 指定微信账号 | 只导出某个账号的数据 |
| 指定会话 | 选择一个或多个联系人、群聊 |
| 指定时间范围 | 只导出某段时间内的聊天 |
| 当前搜索结果 | 只导出当前搜索命中的内容 |
| 已选中的会话 | 从聊天记录页面带入已选会话 |
| 收藏会话 | 只导出收藏的会话 |
| 有附件的会话 | 只导出包含图片、文件、语音或视频的会话 |

默认状态：

- 未选择范围时，主按钮禁用。
- 主按钮文案：请选择导出范围。

选择范围后：

- 右侧预览更新。
- 主按钮变为：下一步，选择导出格式。

指定会话选择器：

- 支持搜索联系人或群。
- 支持多选。
- 支持全选当前搜索结果。
- 显示已选数量。
- 支持移除单个已选项。

右侧预估内容：

- 会话数量。
- 消息数量。
- 图片数量。
- 语音数量。
- 视频数量。
- 文件数量。
- 预计导出大小。

### 12.4 第 2 步：选择导出格式

页面标题：你想导出成什么格式？

格式卡片：

| 格式 | 给用户的说明 | 适合场景 |
| --- | --- | --- |
| HTML 归档 | 可以像网页一样打开查看，保留聊天结构 | 长期保存、自己查看 |
| PDF | 固定版式，方便打印或提交 | 打印、提交材料 |
| CSV | 表格格式，不完整保留媒体 | 数据分析 |
| JSON | 结构化数据，适合开发处理 | 技术迁移或二次处理 |
| Markdown | 轻量文本格式，可放入笔记工具 | 笔记归档 |
| 原始数据库包 | 保留原始结构，不适合普通查看 | 高级备份 |
| 加密迁移包 | 可用于换电脑恢复，需要密码 | 换电脑、长期安全保存 |

默认推荐：HTML 归档。

点击格式卡片后：

- 卡片选中。
- 右侧预览切换为该格式说明。
- 下方显示该格式专属配置。

格式限制提示：

- CSV 不包含完整图片、视频和语音。
- PDF 文件可能较大。
- 加密迁移包必须设置密码。
- 原始数据库包不建议直接分享给他人。

### 12.5 第 3 步：设置导出内容

可配置项：

- 包含图片。
- 包含视频。
- 包含语音。
- 包含文件。
- 包含链接预览。
- 包含系统消息。
- 隐藏手机号和微信号。
- 隐藏文件名。
- 添加水印。
- 生成校验清单。
- 使用密码加密。

交互：

- 每次开关变化后，右侧预估大小立即更新。
- 关闭媒体后，预览中媒体显示为“已省略”。
- 开启脱敏后，预览中手机号、微信号、文件名部分隐藏。
- 开启加密后，显示密码输入框和确认密码输入框。
- 密码不一致时，主按钮禁用。
- 密码强度不足时提示风险，但不一定阻断，除非是迁移包。

推荐默认值：

- HTML：包含图片、语音、文件，关闭视频，关闭脱敏。
- PDF：包含图片，关闭视频和语音，默认分页。
- CSV：只导出文本和基础字段。
- 加密迁移包：全部包含，必须设置密码。

### 12.6 第 4 步：选择保存位置

页面标题：导出的文件要保存到哪里？

保存位置选项：

- 本地文件夹。
- 外接硬盘。

点击“选择保存位置”后：

- 显示完整路径。
- 检查是否可写。
- 检查可用空间。
- 检查是否存在同名文件。

路径正常：

- 显示绿色状态：可以保存。

标题：无法保存到这个位置

主操作：重新选择位置

空间不足：

标题：空间不足
说明：预计需要 8.4GB，当前位置只剩 2.1GB。
主操作：重新选择位置
次操作：减少导出内容

标题：已经有同名文件
选项：

- 自动重命名。
- 覆盖原文件。
- 重新选择位置。

### 12.7 第 5 步：开始导出

点击“开始导出”后：

- 底部任务队列新增导出任务。
- 主按钮变为暂停导出。
- 页面显示当前执行阶段。

导出阶段：

7. 导出完成。

导出中展示：

- 总进度。
- 当前阶段。
- 已处理消息数。
- 已处理附件数。
- 当前文件名。
- 预计剩余时间。

暂停：

- 状态变为已暂停。

取消：

- 主按钮：取消导出。
- 次按钮：继续导出。

完成：

说明：已导出 3 个会话、12,482 条消息、426 个附件。
主操作：打开文件夹

## 13. 恢复管理

### 13.1 页面目标

恢复是高风险操作。

页面必须让用户明确：

- 要从哪个备份恢复。
- 会恢复到哪里。
- 会不会覆盖现有数据。
- 恢复前是否创建当前快照。
- 出错后能否回退。

### 13.2 恢复入口

恢复来源：

- 当前备份位置里的历史备份。
- 外部迁移包。
- 自动备份快照。
- 手动导入包。

每个恢复点展示：

- 创建时间。
- 微信账号。
- 会话数量。
- 消息数量。
- 附件大小。
- 完整性状态。
- 是否加密。
- 来源位置。

### 13.3 恢复步骤

恢复分为 5 步：

1. 选择恢复点。
2. 检查恢复点。
3. 选择恢复策略。
4. 确认影响范围。
5. 开始恢复。

恢复策略：

| 策略 | 说明 |
| --- | --- |
| 只补回缺失内容 | 最安全，不覆盖已有记录 |
| 恢复指定会话 | 只恢复选择的联系人或群 |
| 恢复指定时间范围 | 只恢复某段时间内的数据 |
| 创建新备份位置 | 不影响当前数据 |
| 覆盖当前数据 | 高风险，需要二次确认 |

选择覆盖类策略时：

- 必须显示红色风险提示。
- 必须勾选确认。
- 必须建议创建当前快照。
- 主按钮默认禁用。

确认文案示例：

我知道此操作可能会替换当前已有聊天记录，并已确认需要继续。

完成后显示：

- 恢复会话数。

- 恢复附件数。

## 15. 按钮状态规范

所有按钮必须覆盖以下状态：

| 状态 | 视觉和行为 |
| --- | --- |
| 默认 | 正常可点击 |
| 悬停 | 背景或边框轻微变化 |
| 按下 | 轻微缩放或下沉 |
| 聚焦 | 显示清晰 focus-visible |
| 禁用 | 不响应点击，必须有禁用原因 |
| 加载中 | 显示 spinner，避免重复点击 |
| 成功 | 短暂显示完成文案或对勾 |
| 危险 | 使用红色或浅红背景 |

禁止：

- 点击后没有任何反馈。
- 长任务只弹 Toast。
- 禁用按钮没有原因。
- 危险操作没有二次确认。
- 主按钮文案一直不变。

## 16. Toast 规范

Toast 用于轻量反馈。

适用场景：

- 设置已保存。
- 已复制路径。
- 已加入任务队列。
- 搜索已清空。
- 已打开文件夹。

不适用场景：

- 导入失败详情。
- 导出失败详情。
- 恢复风险说明。
- 长任务进度。

Toast 类型：

| 类型 | 停留时间 |
| --- | --- |
| 成功 | 2 秒 |
| 信息 | 3 秒 |
| 警告 | 5 秒 |
| 错误 | 6 秒，提供查看详情 |

同类 Toast 不应无限堆叠。

## 17. 抽屉规范

抽屉用于查看详情，不打断当前流程。

使用场景：

- 账号详情。
- 会话详情。
- 任务详情。
- 检查报告。
- 错误日志。
- 当前备份位置详情。
- 导出报告。
- 恢复报告。

交互：

- 从右侧滑入。
- Esc 关闭。

## 18. 弹窗规范

弹窗只用于必须打断用户的决策。

使用场景：

- 取消长任务。
- 覆盖已有数据。
- 删除记录。
- 恢复覆盖当前数据。
- 开启或关闭加密。
- 磁盘空间不足且需要用户选择方案。

弹窗必须包含：

- 明确标题。
- 风险说明。
- 影响范围。
- 主操作。
- 次操作。
- 必要时要求勾选确认。

## 19. 空状态规范

所有空状态都必须告诉用户下一步。

场景与文案：

### 19.1 首次使用无备份

标题：还没有备份微信聊天记录
说明：开始第一次备份后，你可以在这里查看、搜索和导出聊天记录。
主操作：开始第一次备份

### 19.2 搜索无结果

标题：没有找到相关内容
说明：请换一个关键词，或清空搜索查看全部内容。
主操作：清空搜索

### 19.3 任务队列为空

标题：当前没有任务
说明：备份、导入或导出开始后，会显示在这里。
主操作：立即备份

### 19.4 导入未选择文件

标题：请选择要导入的文件
说明：我们会先检查文件，不会立刻修改当前数据。
主操作：选择文件

### 19.5 导出未选择范围

标题：请选择要导出的聊天记录
说明：可以导出全部记录，也可以只导出某个联系人或群聊。
主操作：选择导出范围

## 20. 错误状态规范

错误必须可救援。

错误信息包含：

1. 哪里出错。
2. 为什么出错。
3. 影响什么。
4. 用户下一步可以做什么。

示例一：

标题：无法读取备份文件
原因：文件可能被其他程序占用，或你没有读取权限。
影响：当前导入无法继续，已读取内容不会保存。
主操作：重新选择文件
次操作：查看详细日志

示例二：

标题：保存位置空间不足
原因：预计需要 8.4GB，当前位置只剩 2.1GB。
影响：导出任务无法开始。
主操作：重新选择位置
次操作：减少导出内容

示例三：

标题：发现重复聊天记录
原因：当前备份位置里已经有相同账号的部分聊天记录。
影响：继续导入可能产生重复内容。
主操作：只补充缺失内容
次操作：查看重复明细

## 21. 文件系统与权限

涉及本地文件时必须处理：

- 文件不存在。
- 文件被占用。
- 没有读取权限。
- 没有写入权限。
- 外接硬盘断开。
- 网络位置不可达。
- 磁盘空间不足。
- 路径过长。
- 文件名冲突。
- 文件损坏。

交互原则：

- 明确指出哪个路径有问题。
- 不直接让任务失败，能暂停就暂停。
- 用户修复后可继续。
- 需要重新选择路径时提供主按钮。
- 危险覆盖必须确认。

## 22. 键盘与快捷操作

快捷键：

| 快捷键 | 行为 |
| --- | --- |
| Cmd/Ctrl + F | 聚焦搜索 |
| Esc | 清空搜索、关闭抽屉或取消弹窗 |
| Enter | 执行当前主操作 |
| Space | 暂停或继续选中任务 |
| Cmd/Ctrl + A | 当前列表全选 |
| Delete | 删除或移除选中项，需要确认 |
| Cmd/Ctrl + E | 导出选中会话 |
| Cmd/Ctrl + R | 重新扫描 |

优先级：

1. 有确认弹窗时，Esc 取消弹窗。
2. 有抽屉时，Esc 关闭抽屉。
3. 有搜索词时，Esc 清空搜索。
4. 没有特殊状态时，Esc 不做破坏性操作。

## 27. 验收标准

### 27.1 小白可用性验收

- 用户不看说明文档，也能知道第一步点哪里。
- 每个页面都有明确主操作。
- 每个禁用按钮都有原因。
- 每个长流程都有进度和当前步骤。
- 每个失败状态都有下一步操作。
- 专业词不会出现在主流程按钮上。
- 导入和导出流程不需要用户理解数据库概念。

### 27.2 交互完整性验收

- 所有一级导航都能真实切换。
- 导入导出不会锁死其他路由。
- 顶部搜索能真实过滤当前页面。
- 搜索结果有高亮、数量和空状态。
- 任务能暂停、继续、取消和重试。
- 队列任务点击后能查看详情。
- 抽屉关闭后页面状态不丢失。
- Toast 不替代长流程状态。

### 27.3 导入导出验收

导入必须覆盖：

- 来源选择。
- 文件选择。
- 文件检查。
- 警告和错误。
- 保存方式确认。
- 重复策略。
- 附件策略。
- 导入进度。
- 完成、部分完成、失败。

导出必须覆盖：

- 范围选择。
- 会话搜索和选择。
- 格式选择。
- 内容设置。
- 脱敏和加密。
- 保存位置检查。
- 文件冲突处理。
- 导出进度。
- 完成、失败、报告。

### 27.4 视觉和布局验收

- 头部和左侧导航保持稳定。
- 左侧底部当前备份位置不超过 56px。
- 内容区不退回简单卡片堆叠。
- 导入导出呈现专业迁移工具感。
- 按钮、输入框、步骤条、队列、抽屉使用统一组件语言。
- 窗口缩小时不出现文字重叠。
- 状态颜色不过度刺眼，风险信息清楚但不恐吓用户。

## 28. 后续实现建议

建议按以下顺序实现原型，不要一次性重写全部页面：

1. 先修全局框架：路由、侧栏折叠、当前备份位置状态条、全局搜索。
2. 再修基础状态系统：Toast、抽屉、弹窗、按钮 loading、禁用原因。
3. 单独实现导入导出工作台：先状态机，后视觉细节。
4. 回到聊天记录和备份任务：补搜索、选择、详情、任务控制。
5. 最后做全页面回归：检查导入导出没有破坏其他路由。
